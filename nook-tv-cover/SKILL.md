# nook-tv-cover

这个 skill 用来生成 Nook 视频封面。默认不是纯文生图，而是基于参考图的图生图工作流。

本 skill 只处理三类视频封面：

- 长封面：YouTube，`1280x720`
- 短封面：B站，`1146x717`
- 竖封面：抖音、小红书、视频号，`1242x1660`

公众号封面不在本 skill 中处理，后续使用 `nook-WeChat-cover`。

## 必读规则

开始封面任务前，先读取：

- `references/model-setup.md`：确认默认出图通道、开源用户配置方式和失败处理。
- `references/cover-workflow.md`：确认用户引导、常见视频封面规律、风格、尺寸、输出和验收规则。

## 默认资产

- 默认风格参考图：`assets/style/figma-codex-cover-style-reference.webp`
本 skill 不使用公众号 IP 形象，不读取公众号尺寸参考。

## 执行原则

1. 先引导用户上传人物图：真人口播截图、IP 图、白底图、半身图或场景图都可以，但必须说明人物来源和授权。
2. 再确认关键内容：主题、主标题、副标题、英文标签、目标平台、风格和候选数量。
3. 先判断平台：长封面、短封面、竖封面，或三类全部生成。其他封面类型不在本 skill 中处理。
4. 人物图需要具备抠图/分离能力：白底图直接分离，场景图先识别人像主体并弱化或替换背景；必要时生成前先要求模型把人物从背景里抽出。
5. 先确认或推断标题。标题缺失时，可以先生成一版临时标题烟测图，但必须标注为 draft。
6. 同一批次封面必须先锁定一组固定文案：主标题、副标题、英文标签。生成 2-4 张变体时，只能变构图和细节，不能改字。
7. 默认采用图生图：风格参考图 + 人物参考图 + 可选主题素材图。
8. 视频平台人物必须做图生图风格迁移，让人物与背景统一。
9. 同一批次标题透视强度必须一致，默认使用强透视、厚描边、厚阴影标题，不允许一张平面字、一张强透视。
10. 视频封面人物必须是前景，压住标题边缘；人物保持原比例，垂直高度默认只占画布约 65%，头顶上方默认留出约 35% 呼吸空间。
11. 默认风格使用“科技风封面风格”，不要随机切换成赛博、极简、商务蓝或小清新。
12. Nook 本地默认使用 Codex Desktop 内置无 key 出图能力，也就是日志中的 `image_generation_call` / `image_generation_end`。
13. 只有用户明确指定某个模型、工具、CLI 或 API 时，才切换到对应外部通道。
14. 如果当前运行时没有暴露内置出图能力，必须明确说明“当前运行时未暴露 Codex Desktop 内置出图通道”，不要自行改走 API/CLI。
15. 生成后必须把最终图保存到当前项目可找到的位置，不能只留在 Codex 默认生成目录。
16. 不覆盖用户已有资产。需要替换时先确认，否则使用版本号文件名。

## 输出位置

生成图统一放到：

`50_Assets/AI_Images/nook-tv-cover/`

命名建议：

- `video-cover-<topic>-v01.png`
- `video-cover-short-<topic>-v01.png`
- `video-cover-vertical-<topic>-v01.png`

## 不允许

- 不允许生成公众号封面；用户要公众号封面时，转交 `nook-WeChat-cover`。
- 不允许把公众号 IP 人物用于视频封面。
- 不允许把用户私有参考图静默上传到第三方图像服务。
- 不允许把默认出图请求路由到 `image_gen.py`、OpenAI SDK、baoyu-imagine、OpenRouter 或其他 API/CLI。
- 不允许把“当前运行时未暴露 Codex Desktop 内置出图通道”包装成 key、模型权限或 API 问题。
