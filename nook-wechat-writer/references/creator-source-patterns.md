# 创作者来源模式

这份 reference 负责保留 nook 写作方法吸收过的外部创作者价值点。

nook 不是卡兹克或花叔的仿写器。卡兹克和花叔是重要参考样本，但最终目标是把他们身上可复用的内容生产能力，沉淀成自己的写作工作流。

## 目录

- 1. 使用边界
- 2. 卡兹克：活人感、亲自下场和内容张力
- 3. 花叔：流程感、判断系统和交付稳定性
- 4. 两者合流后的 nook 写法
- 5. 不可丢失价值点
- 6. 归因规则

## 1. 使用边界

研究创作者时，先拆方法，不抄表皮。

可以吸收：

- 选题判断方式。
- 内容推进方式。
- 读者关系处理方式。
- 真实细节使用方式。
- 标题和开头的价值判断。
- 审校时保留人味的标准。

不要照搬：

- 口头禅。
- 人设。
- 固定结尾。
- 私人经历。
- 未授权的句子。
- 只有对方账号语境才成立的表达。

如果文章明确研究卡兹克、花叔或任何外部创作者，正文第一次使用对方明确框架或判断时要交代来源。不要把外部方法提前揉进 intro 和开篇，当成自己的原创判断。

## 2. 卡兹克：活人感、亲自下场和内容张力

卡兹克这条线最有价值的不是口头禅，而是“一个真人真的做过、真的看见、真的在判断”。

### 2.1 HKR 选题质检

动笔前用 HKR 看选题是否成立：

- Happy：是否足够有趣、有悬念，能让人想点开。
- Knowledge：是否有信息量，读者看完能学到新东西。
- Resonance：是否戳中情绪，让读者觉得“这说的是我正在经历的事”。

S 级选题通常三项兼备。至少要命中两项才值得进入成稿。只命中一项时，先改角度；零项时不要硬写。

### 2.2 选题交集模型

好选题通常在三个区域交汇：

- 作者真的懂或真的做过的领域。
- 读者正在关心的普遍问题。
- 当下值得讨论的时间节点。

只有专业，容易曲高和寡。只有热点，容易虚。只有读者痛点但没有作者判断，容易变成泛泛建议。三者交汇，文章才更稳。

### 2.3 文章原型

卡兹克内容里有几种可复用原型：

- 调查实验型：亲自下场做一件事，边做边报道发现。核心是过程叙事和层层发现。
- 产品体验型：拿到产品实际使用，带读者一起体验。核心是场景演示和真实感受。
- 现象解读型：从一个有趣现象出发，经过好奇、研究、知识补充，最后给出判断。
- 工具分享型：不是直接推荐工具，而是用个人故事、真实问题和实际效果包住工具。
- 方法论分享型：给出可执行动作，同时承认学习成本、失败点和不确定性。

nook 写作时不要机械套类型。先确认读者价值，再选择最能承载价值的原型。

### 2.4 亲自下场

亲自下场是可信度来源。

优先写：

- 我实际点了哪个按钮。
- 我试了哪个工具。
- 我看到什么界面。
- 我删掉了哪种表达。
- 我在哪一步卡住。
- 我后来怎么改。

不要写：

- “比如有一次”式假经历。
- 过度完整的戏剧化场景。
- 为了证明观点临时捏出来的人物。

没有亲自下场，就诚实说没有。真实的不完整，比虚构的完整更可信。

### 2.5 节奏系统

节奏不是把每句话拆成一行，而是让读者能一路跟着走。

有效做法：

- 围绕主线小幅偏出去，补案例、补知识、补吐槽，再用一句话拉回来。
- 关键判断可以短句单独成段，但不能通篇单句成段。
- 展示多个工具、案例或模型时，用逐一展示法，每个都有发现感和点评。
- 排列案例时采用递进顺序，把最有冲击力的结果放后面。

危险做法：

- 突然偏离主线太远，再硬拽回来。
- 每段都像总结。
- 连续排比制造气势。
- 为了显得有节奏，把普通句子拆碎。

### 2.6 知识顺手掏

知识点要像“聊到这里刚好想起来”，不要像百科条目。

可以先讲当前问题，再自然引入心理学、历史、产品原理、行业背景。知识服务理解，不服务炫耀。

坏写法是“下面我们来介绍这个概念”。好写法是“这块如果不拆一下，后面就容易误解”。

### 2.7 文化升维

文化、历史、哲学参照物可以让文章有余味，但必须从具体问题自然长出来。

判断标准：

- 参照物是否真的解释了当前问题。
- 是否能让读者对原问题理解更深。
- 是否避免了硬拔高度。
- 是否没有替代具体论证。

### 2.8 人物画像法

人物画像可以把数据变成有温度的人。

但人物画像不能伪装成真实采访。没有真实来源时，要写成推测或想象，不要写成“我遇到过一个人”。

好的画像从一个真实数据、真实场景或真实问题出发，用少量细节让读者看见一个具体处境。不要为了煽情堆苦难。

## 3. 花叔：流程感、判断系统和交付稳定性

花叔这条线更重要的是内容生产系统：把选题、材料、结构、标题、发布和复盘拆成稳定流程。

这里保留的是系统价值，不是个人语气。

### 3.1 先有判断，再有表达

文章动笔前先问：

- 读者是谁。
- 读者为什么在乎。
- 这篇解决什么问题。
- 文章交付什么判断、方法或清单。
- 哪些材料服务主线，哪些材料只是作者自己觉得有趣。

如果这些没确认，写得再顺也只是顺滑的废话。

### 3.2 材料取舍

材料不是越多越好。

保留材料时看三点：

- 是否支撑主线。
- 是否提供新信息。
- 是否能让读者更容易理解或行动。

删除材料时也看三点：

- 只是展示作者做了很多功课。
- 只是补齐结构。
- 只是让文章显得更完整但不推进理解。

### 3.3 标题是价值测试

标题不是最后包装。

早期临时标题要用来反查选题是否成立：如果标题写不出对象、痛点、收益、悬念或搜索词，说明文章价值还不清楚。

标题必须接近真实可兑现的价值，不靠夸大、恐吓、虚假冲突或空泛热词。

### 3.4 三轮审校

审校不是润色，而是生产环节。

第一轮看硬伤：事实、链接、工具名、格式、错别字、敏感信息。

第二轮看理解：主线、结构、例子、转场、可交付物。

第三轮看人味：AI 腔、机械排比、假顺滑、没有判断、没有真实细节。

每轮只修本轮问题。不要一次性“优化全文”。

### 3.5 发布交付

公众号成稿不是只有正文。

发布前要准备：

- YAML 标题和摘要。
- 发布标题和发布摘要。
- 标签和搜索关键词。
- 图片标注或图床链接。
- 固定头尾图和尾部引导。
- 字数预告和阅读时间。
- 发布前检查记录。

标题和摘要进入公众号后台时通常需要手动复制，不要指望 Markdown YAML 自动带入。

## 4. 两者合流后的 nook 写法

nook 要把两条线合起来：

- 用花叔式流程保证生产稳定。
- 用卡兹克式真实细节和判断力保证文章像人。
- 用自己的文章数据库校准声音。
- 用 humanizer 子审校去掉机器感。
- 用发布检查保证能进公众号后台。

写作时的优先级：

1. 读者价值。
2. 事实和来源。
3. 作者真实判断。
4. 结构可读性。
5. 标题传播力。
6. 语言人味。
7. 发布可用性。

如果风格和事实冲突，保事实。

如果风格和读者理解冲突，保理解。

如果参考对象和自己的判断冲突，保自己的判断，并说明边界。

## 5. 不可丢失价值点

整合旧版 skills 时，以下价值点必须保留：

- 公众号文章先做价值点和角度确认，再写正文。
- 标题在正文外部，正文从单段 intro 引用块开始。
- 正文自然段之间不要空行，intro 后保留一个空行。
- 正文标题优先使用 1 到 2 层，标题短、口语、不要编号。
- 发布标题接近 30 字，兼顾点击、搜索、真实可兑现。
- summary、publish_summary、intro 三者分工不同，不能互相复制。
- 涉及新产品、新概念、新数据时先做信息挖掘。
- 每篇发布稿必须有真实细节，不能编造亲历。
- 配图是可选动作，完成正文和审校后再询问是否需要。
- 固定头图、尾图和关注引导属于模板物料，不属于每篇正文创作。
- 图片进入公众号后台前最好是公网图床链接。
- 手动发布时从 YAML 单独复制标题和摘要到后台。
- 审校必须有硬伤、理解、人味三层记录。
- 人味审校必须加载 `nook-humanizer-zh-review`。
- 旧版短视频和跨平台改写的核心价值要保留：目标平台决定结构，不是简单压缩或扩写。

## 6. 归因规则

对内使用时，可以把卡兹克和花叔作为风格来源和方法来源。

对外发布时，要遵守三条规则：

- 如果文章只是吸收了通用方法，不必每段都提来源。
- 如果文章使用了对方明确框架、术语或判断入口，首次出现时要标来源。
- 如果文章以研究对方为主题，要让读者清楚区分：哪些是对方方法，哪些是作者自己的提炼。

不要为了显得原创而抹掉来源，也不要为了显得谦虚而让来源抢走文章主价值。
