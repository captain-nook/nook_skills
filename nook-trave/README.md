# nook-trave

自驾旅行路书规划 skill。当前首个完整样例是“成都出发，新疆全程自驾，不超过 26 天”，用于跑通从意图收敛到竖屏路书交付的完整流程。

## 能力范围

- 开场仪式：提示用户接下来只需回答 8 个简单问题。
- 单问单答：一次只问一个问题，降低长交互压力。
- 路线规划：生成路线骨架、每日节奏、住宿点和体验重点。
- 地图过程可见：地图采集或生成时，要把当前进度同步给用户。
- 静态地图：支持高德静态总览图和每日底图。
- 里程核算：支持通过高德驾车接口核算每日距离和时长。
- 路书交付：输出适合手机阅读的竖屏 HTML 路书样张。

## 目录

- `SKILL.md`：Codex skill 主入口。
- `references/`：工作流、路书规范、工具策略。
- `scripts/`：路书渲染、静态地图、里程核算和每日地图生成脚本。
- `data/`：新疆长线自驾样例数据和路线核算数据。
- `output/`：样张 HTML、总览地图和每日地图。
- `assets/roadbook-template/`：路书模板资产。

## 地图 key

下载高德静态地图或调用高德驾车接口时，需要本地提供 key：

```powershell
$env:AMAP_WEB_SERVICE_KEY='your-key'
```

key 不能写入仓库。没有 key 时，脚本仍应输出可检查的地图 URL 或提示缺失原因，让流程继续向下跑。
