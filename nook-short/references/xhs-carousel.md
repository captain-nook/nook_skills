# Xiaohongshu Carousel

Plan cover and page structure before writing text.

## Default Page Roles

```text
1. cover: object + result / pain + save reason
2. problem: why this matters
3. overview: map / flow / framework
4. detail 1: module / step / pitfall
5. detail 2: module / step / pitfall
6. who it is for: audience and use case
7. save / material / next post: weak in-platform continuation
```

Use 6-9 pages by default.

## Checks

- Cover can be understood as a thumbnail.
- There is a clear save reason.
- At least 3 searchable keywords are present.
- Real screenshots, flowcharts, or interface evidence are preferred.
- Do not split a short-video script into carousel lines.
- Do not use strong off-platform lead generation.

## Text Density

Prefer short page copy. Each page should carry one task. Use platform samples to calibrate words per page when available.
