# Writing Templates

Fill slots before drafting. If the slots do not fit, change the model.

## Douyin Result

```text
花了 X 天 / 折腾了 X 时间，我用 AI 搭了一套【系统/工作流】，直接【省时间/跑通结果/改变动作】。
```

Rules:

- First show result.
- Do not teach the full method.
- Use one visible proof.
- End with in-platform collection/profile path, not mixed material claims.

## Douyin Pitfall

```text
做【事情】的朋友注意，90% 的人用 AI 都踩了这个坑。
```

## Douyin Viewpoint

```text
我敢说，再过半年，只会用 Prompt 的人根本玩不转 AI。
```

## Xiaohongshu List

```text
AI 做【场景】必备的 X 个工具，建议收藏。
```

## Xiaohongshu Pitfall

```text
折腾一周【项目】，终于把最容易踩的 X 个坑踩明白了。
```

## WeChat Channels Pain

```text
很多做【业务/内容/团队】的人，想靠 AI 提效，但不知道从哪下手。
```

## WeChat Channels Case

```text
上周帮一个做【业务】的朋友搭了套 AI 系统，现在每天省【具体时间/动作】。
```

Do not invent fake cases. If no real case exists, use process or viewpoint model.

## Human Check

Remove or reduce:

- “不是……而是……” unless it makes a real distinction.
- “真正的 / 本质上 / 这意味着 / 综上所述 / 值得注意的是”.
- Long report-like transitions.
- Consecutive rhetorical questions.
- Fake numbers and fake users.
