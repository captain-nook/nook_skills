# image2.0 直出小红书封面合同

本文件用于修正 `nook-cover` 的默认判断：小红书爆款封面优先走整图生成，而不是优先走 HTML。

HTML 可以稳定中文和排版，但当前阶段很难自然生成以下关键质感：

- 真实人物大占比但不居中。
- 夸张表情和手势。
- 真实撕纸纤维、纸张毛边、胶带透明质感。
- 非标准字体的大字形变。
- 文字、人物、装饰元素之间的真实遮挡关系。
- 手绘星星、箭头、划线、短线与画面融合。

因此，小红书封面默认路线应为：

```text
主题与文案 -> 选择视觉系统 -> 写 image2.0 整图提示词 -> 生成整图 -> QA -> 必要时 HTML / 后处理补字
```

HTML 路线降级为：

```text
结构验证 / 中文兜底 / 简单信息图 / 没有图片模型时的保底方案
```

## 高质量样张的可执行特征

image2.0 直出封面必须追求以下特征：

- 画面 70% 以上是强视觉：人物、大字、撕纸背景共同占满画面。
- 主标题占画面 45%-60%，不是顶部小标题。
- 标题是非标准字体：粗、扁、夸张、略有透视或错位。
- 标题有三层效果：红 / 黑高对比字色、厚白描边、黑色硬阴影。
- 标题分 2-3 行，行间紧密，局部重叠。
- 人物在右侧或下侧偏位，占画面 35%-55%，不能像证件照居中。
- 表情必须夸张：惊讶、瞪眼、张嘴、指向、拿物。
- 背景必须有红黄高饱和撕纸、网格纸、纸张纹理。
- 装饰元素必须参与叙事：星星、箭头、手绘短线、贴纸、便签、胶带。
- 小副标题放在黄色便签或纸片上，不做普通 UI 卡片。

## image2.0 提示词骨架

```text
生成一张小红书爆款封面，竖版 3:4。
主题：{主题}
主标题：{主标题，2-3 行}
副标题：{副标题}

视觉要求：
红黄高饱和撕纸背景，真实纸张毛边，网格纸纹理；
超大中文标题，占画面 45%-60%，非标准粗体字形，红黑高对比，厚白描边，黑色硬阴影，轻微透视和错位堆叠；
人物在画面右侧，不居中，占比很大，表情夸张，张嘴惊讶，手指指向标题或便签；
加入手绘星星、箭头、短线、胶带、黄色便签、划线；
整体像真实小红书爆款封面，不像 PPT，不像普通海报，不像 UI 卡片。

中文文字必须准确：
主标题：{主标题}
便签文字：{便签文字}
```

## QA

通过条件：

- 缩略图一眼先看到大标题。
- 人物和标题形成强关系。
- 有明显小红书装饰语言。
- 不是普通生活照片加字。
- 不是 HTML UI 卡片感。
- 中文可读且核心文字正确。

如果 image2.0 整图中文轻微翻车，但构图和风格很好，可以保留为底图，再走 HTML / 后处理重做文字层。
