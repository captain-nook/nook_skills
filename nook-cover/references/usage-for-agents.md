# Agent 使用说明

当用户要求生成小红书封面时，按下面流程执行。

执行小红书封面成品生成时，必须优先读取 `references/image2-direct-cover-contract.md`。当前默认路线是 image2.0 整图直出，HTML 只作为中文兜底、结构验证、批量换字或后处理文字层。

只有在明确选择 HTML 路线时，才读取 `references/v05-html-visual-contracts.md` 和 `references/v05-template-map.md`。四套视觉系统不是审美描述，而是 HTML 可执行的构图、文字效果和辅助元素合同。

## 1. 先判断路线

默认路线：

```text
有 Codex Desktop 内置 image2.0 时，直接出整张封面候选
```

使用外部工具时：

- 快速草图 / 免费测试 / 亚洲人物：`nook-zimage`
- 中文海报 / 中文视觉 / 中文文字测试：`nook-qwen-image`
- 高质量成品 / 图生图 / 参考图保持：`nook-image2-gpt` 或内置 image2.0

如果当前运行时没有 Codex Desktop 内置 image2.0，也没有配置外部工具，不要假装完成了真实封面生成。只能输出 HTML / Playwright 排版证明，并明确说明缺少真人、产品或场景主视觉。

## 2. 生成 brief

至少要确定：

- 平台：小红书。
- 标题。
- 主题。
- 视觉系统：人脸冲击封面、手账拼贴种草、产品截图主视觉、近大远小对比封面，四选一。
- 是否有人物。
- 是否有真实主视觉来源：Codex image2.0、外部原子工具、用户素材，三者至少要有一个；否则只能做 HTML 排版证明。
- 是否允许模型直接生成中文。
- 若中文不准，是否转 HTML / 后处理标题层。

生成前必须输出内部结构化 brief，至少包含：

- `visual_system`
- `layout_contract`
- `title_lines`
- `html_primitives`
- `composition_contract`
- `qa_contract`

没有结构化 brief，不允许直接渲染。不要只根据“暖色治愈”“人脸冲击”这类词直接出图。

如果用户选了“人脸冲击封面”，必须补问或自动推断：

- 人物动作：指向标题、展示产品/食物、夸张反应、手持对比卡。
- 主标题拆分：2-3 行巨大标题，缩略图可读。
- 前景爆点：数字、挑战、避坑、对比、承诺。

减脂 / 食谱主题不要默认做生活照。必须把菜品、人物、标题都做成小红书贴纸封面关系。

## 3. 输出规则

输出必须进入用户指定目录。没有指定时，进入当前项目的 `output/`。

过程图和候选图不要删除。失败图也可以保留，用于后续复盘。

## 4. QA

必须逐张检查：

- 中文是否正确。
- 标题是否足够大。
- 人物是不是完整半身 / 三分之二身，而不是头像。
- 标题有没有遮脸。
- 撕纸、贴纸是否自然。
- 是否像小红书，而不是 PPT。
- 是否像 App 菜单或普通生活照；如果像，判定失败。
- 主标题是否占据至少 35% 视觉权重；如果标题只在顶部，判定失败。
- 是否一眼能看出属于所选视觉系统；如果看不出，判定失败，即使中文正确。

如果失败，不要只改颜色。要重新选择构图机制或提示词。
