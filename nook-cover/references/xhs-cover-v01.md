# 1 小红书封面 V0.1

## 1.1 目标

根据一句主题或标题，生成一张小红书封面 / 首图。

## 1.2 输出

默认画布：

```text
1080x1440
```

第一版使用 3:4。

## 1.3 Brief Schema

机器字段可以保留英文枚举，面向用户的说明和成果必须使用中文。

```json
{
  "platform": "xiaohongshu",
  "canvas": {
    "width": 1080,
    "height": 1440,
    "ratio": "3:4"
  },
  "topic": "",
  "title": "",
  "title_lines": [],
  "subtitle": "",
  "audience": "",
  "content_type": "tutorial | opinion | tool | case | lifestyle | product",
  "visual_goal": "click | save | trust | taste",
  "visual_system": "life-aesthetic | proof-signal | hook-title",
  "layout_family": "",
  "palette": "",
  "asset_strategy": "none | zimage_background | image2_main_visual | user_asset",
  "text_safe_zone": "",
  "qa": []
}
```

中文标题默认允许显式指定 `title_lines`。不要完全依赖自动切字，尤其是 8-14 字的标题，错误切分会直接破坏封面语义。

## 1.4 视觉系统

V0.1 只保留三套视觉系统。在这三套能稳定出图前，不继续扩展。

### 生活审美型

适合需要审美感、生活方式气质、信任感或轻柔个人表达的内容。

借鉴 Captain 的 Still Paper 思路，但保持更简单：

- 安静背景。
- 大而清楚的标题。
- 温暖或清新的配色。
- 明确文字安全区。
- 少量标签。

### 证据干货型

适合需要证明感、教程价值、工具逻辑、工作流、截图、数字或对比的内容。

借鉴 Captain 的 Signal Proof 思路：

- 证据块。
- 小标签。
- 截图或信息面板。
- 清楚层级。
- 重结构，少情绪。

### 强钩子标题型

适合需要信息流第一眼点击力的封面。

这是小红书封面额外需要的一套：

- 标题是主视觉。
- 背景保持简单。
- 一组强对比色。
- 1-2 个辅助标签。
- 缩略图尺寸下仍然可读。

## 1.5 初始布局族

### 干净标题背景

适合需要审美感和清晰度的封面。

- 大标题放在上方或中部安全区。
- 背景图或材质负责提供情绪。
- 标签尽量少。
- 文字和背景必须强分离。

### 物件加标题

适合有产品、工具、书、截图或可识别物件的内容。

- 主物件占据下半部或一侧。
- 标题放在另一侧安全区。
- 允许 1-2 个小标签。

### 信息钩子封面

适合教程、方法、清单、对比内容。

- 大标题。
- 2-3 条短要点或数字。
- 清楚的卡片式信息块。
- 背景保持简单。

## 1.6 素材规则

`nook-cover` 不封装出图工具本体，只定义路由。三套出图工具独立安装、独立配置、独立保存 `.env`。

用 `nook-zimage` 做低成本背景探索和批量草稿。

用 `nook-qwen-image` 做中文海报、中文封面、需要中文文字参与画面的出图任务。

用 `nook-image2-gpt` 做高质量主视觉或图生图，尤其是需要参考图保持、风格迁移或更强视觉质感的任务。

不要依赖出图模型生成最终中文文字。最终标题和文字必须由 HTML 渲染。

V0.1 中，`asset_strategy: "none"` 是合法策略。先用 CSS 背景证明封面链路，再加入出图工具。

默认出图路由：

```text
测试 / 批量 / 低成本背景 → nook-zimage
中文海报 / 中文封面 / 模型内文字 → nook-qwen-image
高质量主视觉 / 图生图 / 参考图保持 → nook-image2-gpt
HTML 排版成品 → nook-cover 自己控制文字和截图导出
```

如果三套工具都未配置，仍然可以用 CSS / HTML 占位底图跑通封面结构；这条路线用于安装验证和低风险演示。

## 1.7 QA

接受成品前必须检查：

- 手机缩略图尺寸下标题仍然可读。
- 文字不能融进背景。
- 背景不能有生成的伪文字。
- 主体不能和标题互相打架。
- 画面只有一个明确焦点。
- 成品像可发布的小红书首图，而不是 PPT 页面。

## 1.8 V03 更新

如果用户要求“小红书感”“爆款感”“不像 PPT”，不要继续使用 V0.1 的规整卡片式封面。

优先读取：

```text
references/xhs-poster-v03-visual-systems.md
```

V03 已经把视觉系统升级为：

- 甜橙贴纸爆款。
- 绿调手账种草。
- 蓝黑科技榜单。
- 黑橙力量冲击。

V03 的判断标准是：标题必须像贴纸或刷字一样压进画面，允许倾斜、错位、描边和夸张装饰。不能只做横平竖直的圆角矩形卡片。
