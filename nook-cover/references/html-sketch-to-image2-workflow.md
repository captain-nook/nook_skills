# HTML 草图到 image2.0 图生图工作流

本路线用于解决一个核心矛盾：

- HTML 擅长控制布局、文字层级、信息位置和构图草图。
- image2.0 擅长生成真实人物、非标准字形、撕纸质感、手绘装饰和整体视觉融合。

因此，小红书封面推荐路线改为：

```text
主题与文案
-> 选择视觉系统
-> HTML 生成低保真布局草图
-> image2.0 以草图为参考图进行图生图重绘
-> QA
-> 必要时 HTML / 后处理补中文
```

## HTML 草图的定位

HTML 草图不是成品，不要求漂亮。它只负责传达：

- 主标题大概在哪。
- 主标题分几行。
- 人物在左侧、右侧、下方还是偏位。
- 便签、箭头、星星、撕纸、贴纸的大致关系。
- 画面重心和留白比例。

HTML 草图必须避免：

- 把自己做成最终 UI 卡片。
- 用规整按钮、菜单、PPT 卡片冒充小红书风格。
- 花很多力气模拟真实人物和真实撕纸。

## image2.0 图生图提示词骨架

把 HTML 草图作为参考图，然后使用下面的提示词：

```text
请参考这张 HTML 草图的布局关系，但不要保留网页感、UI 感和规整矩形感。
把它重绘成真实小红书爆款封面，竖版 3:4。

主题：{主题}
主标题：{主标题}
副标题 / 便签文字：{便签文字}

必须保留草图中的构图关系：
- 主标题仍在草图对应区域，分 2-3 行，占画面 45%-60%。
- 人物仍在草图对应侧，不居中，占画面 35%-55%。
- 便签、箭头、星星、撕纸仍围绕标题和人物组织。

请重绘为真实封面效果：
- 红黄高饱和撕纸背景，真实纸张毛边和纸张纹理。
- 超大中文标题，非标准粗体字形，红黑高对比，厚白描边，黑色硬阴影，轻微透视和错位堆叠。
- 人物表情夸张，张嘴惊讶，手指指向标题或便签，人物不居中。
- 加入手绘星星、箭头、短线、胶带、黄色便签、红色划线。
- 整体像真实小红书爆款封面，不像 PPT，不像普通海报，不像网页 UI。

中文文字必须准确：
主标题：{主标题}
便签文字：{便签文字}
```

## QA

通过条件：

- 一眼看出 image2.0 重绘后不是 HTML 截图。
- 大标题比 HTML 草图更像手工设计字，而不是普通系统字体。
- 人物和标题有真实遮挡、光影或互动关系。
- 撕纸、胶带、星星、箭头有真实质感。
- 保留了 HTML 草图的构图意图。
- 中文核心文字可读。

失败条件：

- 仍然像网页截图。
- 仍然像 PPT 卡片。
- 人物太小或居中。
- 标题只是普通字体加描边。
- 丢失草图里的核心信息层级。

## Agent 执行要求

如果运行时具备 image2.0 图生图能力：

1. 先用 HTML renderer 输出草图 PNG。
2. 把草图 PNG 作为参考图交给 image2.0。
3. 提示词中明确“参考布局但重绘真实封面，不保留网页感”。
4. image2.0 产物为主结果。
5. HTML 草图只作为过程文件保存。

如果运行时没有 image2.0 图生图能力：

- 可以先交付 HTML 草图和 image2.0 提示词包。
- 不要把 HTML 草图说成最终小红书封面。
