# 小红书 Poster V04 视觉系统

V03 已经证明了“图片底图 + HTML 叠字”能跑通，但视觉仍有明显问题：撕纸太规整、标题遮住人脸、标题不够大、四套风格差异不够、输出目录过碎。

V04 的目标是修正这些问题。下一轮不要继续沿用 V03 的统一模板逻辑。

核心判断：

- 标题必须是第一视觉，主标题区域建议占画面高度 22%-38%，爆款标题可到 45%。
- 人脸是禁遮挡区。标题、撕纸、贴纸、徽章不能遮住眼睛、鼻子、嘴巴和完整脸部轮廓。
- 小红书感来自“画面化文字”，不是文字框：标题要倾斜、错位、描边、压边、刷底、局部遮挡背景，但不能挡脸。
- 小红书爆款标题常常有透视感：斜着压下来、贴在前景卡片上、数字近大远小、硬阴影制造厚度。
- 撕纸必须有毛边、缺口、凹凸、阴影、轻微旋转。禁止规则多边形冒充撕纸。
- 四套系统必须是构图机制不同，不只是换颜色。
- 输出成果必须集中在一个输出目录，文件名用标题或案例名区分，不再每张图一个子文件夹。

V04-2 路线修正：

- 如果目标是快速接近真实小红书封面，优先用 image2.0 生成整图候选来确认视觉方向。
- HTML 渲染器只有在需要稳定复刻中文字、批量替换标题、或控制可编辑层时再介入。
- 四套风格不能只改文字颜色、描边和背景。每套必须同时改变主体占比、标题占比、人物与标题关系、材料语言、装饰元素和构图动势。

## 0. 同类封面系统参考得到的判断

同类封面生产系统的价值不在“有很多风格名字”，而在于把构图关系说清楚：

- 人物居中，标题压在人物背后。
- 人物侧置，另一侧全给标题。
- 产品或截图占 60%-70%，人物只做引导。
- 对比卡片近大远小，制造透视冲击。
- 正面对视时，文字围绕脸部，不遮眼睛和嘴巴。
- 局部出镜时，产品或标题是主角，人物只是见证。

下一版要采用这种“构图先行”的方法。不要先定义颜色主题，再把所有东西塞进同一套版式。

## 1. 人脸冲击封面

适合副业、减脂、穿搭、成长、情绪钩子。

视觉语言：

- 人物脸部或半身占画面 45%-65%，表情要明显。
- 人物必须有动作：指向标题、举起食物、展示前后对比、拿便签或做夸张反应。平静生活照不合格。
- 标题必须很大，放在脸部上方、侧边或背后，不遮五官。
- 标题用白描边、硬阴影、倾斜排版；允许部分字压到头发边缘，但不能压眼睛。
- 装饰用星星、箭头、勾选、云朵气泡，密度中高。
- 适合暖色和高饱和撞色。

避免：

- 任何大色块遮脸。
- 标题太靠上导致只有额头或头发露出来。
- 小标题过多，削弱第一眼标题。
- 底部三等分按钮、整齐菜单、App 首页式布局。
- 真实照片占满画面，设计层只是薄薄一层标题。

减脂 / 食谱主题专项规则：

- 菜品要变成贴纸化前景素材，而不是规整菜单按钮。
- 必须有一个强爆点：`30天挑战`、`吃饱也能瘦`、`不挨饿`、`一周不重样`。
- 标题应拆成 2-3 行巨大字，如 `减脂30天 / 家常菜谱 / 不挨饿`，字号必须压过人物和菜品。
- 背景厨房可以有，但不能变成安静生活照。要加入撕纸、箭头、对比卡、手写圈画。

## 2. 手账拼贴种草

适合探店、护肤、书单、生活方式、工作台、收藏型内容。

视觉语言：

- 主体可以是食物、护肤品、桌面、咖啡，主体占画面 45%-60%。
- 标题不是一整张大白纸，而是多块不规则纸片、胶带、便签、照片层叠。
- 撕纸边缘必须不规则：毛边、锯齿、凹凸、透明阴影、不同旋转角。
- 标题可以手写感，但必须足够大，手机缩略图能读。
- 装饰用虚线、箭头、手绘心形、手绘太阳、小贴纸。

避免：

- 一块规整大白纸盖住半张图。
- 纸片形状过直，像普通多边形。
- 所有小纸条同角度、同阴影。

## 3. 产品截图主视觉

适合 AI 工具、提示词、效率、程序员、产品教程、软件方法。

视觉语言：

- UI 截图、工具图标、产品画面占 55%-70%，成为主视觉。
- 人物只做引导，体量 20%-35%，可以指向截图或看向截图。
- 标题压在截图边缘或截图后方，不要做 SaaS 官网式信息面板。
- 可用霓虹、斜切框、编号标签，但信息块数量要少。
- 适合深色科技，也可以做高饱和纯色背景。

避免：

- 一堆规整列表和指标框。
- 标题不够大，被截图和工具列表抢走。
- 背景过暗导致像技术海报，不像小红书封面。

## 4. 近大远小对比封面

适合前后对比、好坏对比、避坑、逆袭、挑战、复盘。

视觉语言：

- 前景对比卡片或核心道具必须近大远小，形成透视冲击。
- 人物可在中景，手持或推出前景卡片。
- 标题可以在人物背后或上方，大而斜，不能遮脸。
- 适合黑橙、红黄、蓝黄等强对比。
- 装饰用粗箭头、刷痕、爆炸星、斜切色带。

避免：

- 只有普通大标题，没有前后对比关系。
- 对比卡片太小，近大远小不明显。
- 行动感不够，像普通励志海报。

## 5. 人脸安全区规则

所有含人物封面必须先判断人脸位置。

最低规则：

- 眼睛、鼻子、嘴巴和脸部外轮廓为一级禁区。
- 主标题不得覆盖一级禁区。
- 副标题、标签、贴纸不得覆盖眼睛和嘴巴。
- 如果无法识别人脸，默认把画面上半部中间 35%-70% 宽度设为禁区，避免标题压头。
- 标题可以压头发、肩膀、衣服、背景，不压脸。

## 6. 常见失败模式

看到以下情况，应判定失败并重新生成 brief 或提示词：

- 像菜谱 App、健康管理 App、课程海报，而不是小红书封面。
- 标题只在顶部，画面主体是一张普通照片。
- 底部出现三个或多个整齐按钮，像导航菜单。
- 人物没有表情和动作，只是自然生活照。
- 主标题面积低于画面 25%，缩略图不够冲击。
- 装饰元素只是角标和小标签，没有贴纸、箭头、撕纸、描边、硬阴影。

后续工程实现时，至少要允许 brief 手动指定：

```json
{
  "safe_zones": {
    "face": { "x": 0.38, "y": 0.16, "w": 0.30, "h": 0.28 },
    "title_allowed": ["top-left", "top-right", "bottom-band", "behind-person"]
  }
}
```

## 6. 撕纸与贴纸规则

禁止用规则 polygon 直接当撕纸。

撕纸至少满足三条：

- 边缘有 8 个以上不规则凹凸点。
- 有轻微颗粒或毛边纹理。
- 有贴纸投影，投影方向和纸片旋转一致。
- 多纸片之间角度不同，不能平行。
- 可以用 SVG filter、clip-path 多点扰动、PNG 纸张素材或 canvas 噪声生成。

如果 HTML 做不出自然毛边，就优先使用素材化纸片，不要硬做规整假撕纸。

## 7. 输出目录规则

后续所有样张输出到一个目录，不再每张图一个子文件夹。

推荐：

```text
output/xhs-poster-v04/
├── 001_下班后搞副业.png
├── 002_减脂30天瘦了10斤.png
├── 003_这5个AI工具改变了我.png
└── contact-sheet.jpg
```

HTML 调试文件如需保存，集中到：

```text
output/xhs-poster-v04/html/
```

但正式成果图必须平铺在同一个输出目录。

## 7.1 文字透视规则

V04 必须把文字透视纳入 brief。

推荐字段：

```json
{
  "title_perspective": {
    "mode": "top-slam | side-push | card-perspective | flat-sticker",
    "rotate": -6,
    "skew": -8,
    "perspective": 900,
    "scale_front": 1.08
  }
}
```

使用原则：

- 大数字优先做近大远小，例如“30天”“10斤”“5个”。
- 标题贴在前景卡片上时，卡片和文字必须同角度透视。
- 透视不能牺牲可读性，25% 缩略图仍要看清。
- 透视文字优先服务冲击力，不作为每张图的固定装饰。

## 9. QA

验收时先问六件事：

- 第一眼像不像小红书封面。
- 主标题是不是足够大，是否是第一视觉。
- 有没有遮住人脸，尤其是眼睛、鼻子、嘴巴。
- 撕纸、贴纸、刷底是否自然，不能像规整 PPT 形状。
- 标题在手机缩略图里是否可读。
- 字是否有倾斜、描边、刷底、贴纸感，而不是端正 PPT 标题。
- 同一批图是否能看出 4 套构图机制不同，而不是同一模板换皮。
