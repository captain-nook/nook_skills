# 小红书 V04-2 生产流程

V04-2 的核心修正：先拿到真实的小红书封面感，再考虑可编辑和可复刻。

不要从规整 HTML 模板开始。HTML 固定模板容易把小红书封面做成 PPT 页面。

## 默认路线

```text
输入主题 / 标题
→ 判断内容类型
→ 选择 4 套视觉系统
→ 判断运行时是否有可用图像通道
→ 有图像通道：生成真实主视觉候选
→ 无图像通道：仅生成 HTML 文字排版证明，并明确说明缺少真人 / 产品 / 场景主视觉
→ QA 检查
→ 必要时用 HTML / 后处理稳定中文标题层
→ 输出 PNG
```

图像通道包括：

- Codex Desktop 内置 image2.0。
- `nook-image2-gpt`。
- `nook-qwen-image`。
- `nook-zimage`。
- 用户提供的照片、产品图、截图或背景素材。

ZCode、Claude Code、Cursor 等运行时如果没有暴露 Codex Desktop 内置 image2.0，不要声称已经使用 image2.0。应优先检查外部原子工具是否配置；若未配置，先提醒用户当前只能跑 HTML 稳定文字层。

## 四套视觉系统

### 1. 人脸冲击封面

适用：

- 副业
- 成长
- 普通人经验
- 情绪钩子
- 职业转型

提示词要点：

- 真人半身或三分之二身，不能是头像。
- 人物占 45%-55%。
- 大标题占 35%-45%，必须第一眼读到。
- 标题厚描边、贴纸阴影、轻微透视。
- 装饰：星星、箭头、便利贴、手写标签。

内部别名：人物冲击贴纸风。

### 2. 手账拼贴种草

适用：

- 探店
- 书单
- 护肤
- 生活方式
- 收藏清单

提示词要点：

- 真实撕纸毛边、纸张纹理、胶带、便签。
- 标题写在大块撕纸上，但必须可读。
- 场景要有生活细节，不能只是一张空背景。
- 可读性优先于手写感。

内部别名：撕纸手账种草风。

### 3. 产品截图主视觉

适用：

- AI 工具
- 教程
- 效率工作流
- 软件方法
- 内容生产流程

提示词要点：

- 真人讲解者 + 电脑 / 屏幕 / 科技抽象背景。
- 标题必须大，不能被界面细节抢走。
- 允许霓虹、斜切框、警示贴、箭头。
- 禁止大量不可读 UI 假字。

内部别名：科技教程爆款风。

### 4. 近大远小对比封面

适用：

- 健身
- 自律
- 逆袭
- 前后变化
- 避坑

提示词要点：

- 黑橙高对比。
- 强斜线构图和动势。
- before / after 或对比叙事。
- 人物完整可识别，不能只剩头或局部身体。

内部别名：黑橙冲击对比风。

## QA

每张图必须检查：

- 中文标题是否正确。
- 标题是不是第一视觉。
- 人物是否完整，不是头像。
- 文字有没有遮住眼睛、鼻子、嘴巴。
- 四套风格是否真的不同，而不是换颜色。
- 缩略图下是否能读出标题。
- 画面是否有有效信息密度，不是大片死留白。

## 失败处理

如果中文误写：

- 先保留图作为视觉候选。
- 再用 HTML / 后处理重建标题层。

如果人物变头像：

- 重新生成，提示词必须写明“upper body from waist to head / visible torso and hands / not a headshot”。

如果像 PPT：

- 禁止继续修边框颜色。
- 重新选择视觉系统和构图关系。
