# 触感软拟态卡片风格规范

版本：v0.2  
基准样张：`demo.png`  
基准尺寸：`1546 x 1740 px`

## 风格定义

“触感软拟态卡片风”是一种用于知识卡片、教程封面和步骤说明图的高触感界面风格。它结合了新拟物的实体按钮、粗黑描边的漫画感、硬阴影带来的卡片厚度，以及 macOS 窗口/代码面板带来的软件工具语境。

画面必须让人感觉：元素是可点击、可按压、有厚度的实体 UI，而不是平面海报。

本规范不是固定模板。`demo.png` 只提供风格 DNA 和初始数值，后续允许文字数量、按钮数量、卡片比例、构图密度变化；但所有变体必须共享粗黑描边、硬阴影、实体按钮、高对比色、软件窗口隐喻和低可见度点阵底纹。

## 坐标与单位

- 默认坐标系：左上角为 `(0, 0)`，单位为 px。
- 本规范所有像素值基于 `1546 x 1740 px` 基准画布。
- 允许等比缩放；缩放后所有尺寸、描边、圆角、阴影偏移同步乘以缩放系数。
- 如果使用其他比例，优先保持宽高比接近 `0.889:1`，即 `1546 / 1740`；但允许在 `0.62-1.15` 之间变化，用于长图、方图、横版卡片或视频画面。
- 所有绝对值都可以换算为比例值：`比例值 = 像素值 / 画布宽度`，纵向间距可按画布高度换算。

## 动态布局原则

- 固定的是视觉基因，不是固定坐标。
- 卡片比例可以变化，但主卡片必须是画面最大实体容器，占画布宽度 `72%-88%`。
- 内容区应采用纵向栅格：顶部窗口语义区、主标题区、正文/信息区、行动按钮区。
- 文字多时，优先增加卡片高度或减少字号，不要削弱描边、阴影和按钮厚度。
- 按钮数量可为 `0-4` 个；有按钮时至少一个使用黄色或红色强调色。
- 画面必须保留至少两个风格锚点：粗黑描边、硬阴影、点阵卡片底纹、macOS 三色点、实体按钮、鼠标指针、代码面板。
- 如果是纯知识卡、没有按钮，也要通过粗描边标题块、标签块或状态点保持“软件卡片”感。

## 画布

- 默认画布：`1546 x 1740 px`。
- 方向：竖版。
- 背景色：近白冷灰，建议 `#F7F9F8`，允许范围 `#F4F7F6` 到 `#FAFBFA`。
- 背景不能使用明显渐变、纹理、噪点、图案或大面积装饰光斑。

## 主卡片

- 主卡片位置参考：左边界约 `104 px`，上边界约 `73 px`。
- 主卡片前景可视宽度约 `1230-1240 px`，高度约 `1580-1590 px`。
- 主卡片占画布宽度约 `79.5%-80.5%`，占画布高度约 `90.8%-91.5%`。
- 主卡片填充色：`#FFFFFD` 或 `#FEFEFE`。
- 主卡片表面必须添加低可见度点阵底纹：
  - 点阵颜色：`#DADFDA` 或黑色 `#010101` 的 `4%-7%` 不透明度。
  - 点直径：画布宽度的 `0.14%-0.24%`，基准图中约 `2-4 px`。
  - 点间距：画布宽度的 `1.6%-2.4%`，基准图中约 `25-37 px`。
  - 点阵只能若隐若现，不能影响文字可读性；标题和正文后方可降低到 `0%-3%`。
- 主卡片描边色：`#010101` 到 `#0C0C0C`。
- 主卡片描边厚度：`13-18 px`。
- 主卡片圆角半径：约 `85-95 px`。
- 主卡片右下方必须有实体阴影/厚度层：
  - 阴影主色：`#0B0B0B` 到 `#111111`。
  - 水平偏移：`34-44 px`。
  - 垂直偏移：`38-52 px`。
  - 模糊半径：`0-4 px`，推荐 `0 px`。
  - 扩展半径：`0 px`。
  - 阴影不透明度：`85%-100%`。
  - 阴影边缘必须锐化，不能形成软雾状投影。

## 色彩规范

### 基础色

| 用途 | 推荐值 | 允许范围/备注 |
| --- | --- | --- |
| 背景 | `#F7F9F8` | 近白冷灰 |
| 主卡片填充 | `#FFFFFD` | 可用 `#FEFEFE` |
| 点阵底纹 | `#DADFDA` | 或 `#010101` / `5.5%` opacity |
| 文字/描边 | `#010101` | 允许到 `#111111` |
| 深阴影 | `#0B0B0B` | 允许到 `#151515` |
| 代码面板深色 | `#272935` | 只作为背景语境，不抢主视觉 |

### 强调色

| 用途 | 采样值 | 规范值 |
| --- | --- | --- |
| 黄色主按钮 | `#FDC703` | `#FDC703`，允许 RGB 每通道偏差 `±12` |
| 红色主按钮 | `#E03A10` / `#DB350D` | `#E03A10`，允许 RGB 每通道偏差 `±14` |
| macOS 红点 | `#F7524E` | 直径 `46-48 px` |
| macOS 黄点 | `#FDC528` | 直径 `46-48 px` |
| macOS 绿点 | `#28C959` | 直径 `46-48 px` |
| 右侧状态绿点 | `#28C959` | 与 macOS 绿点保持一致 |

## 按钮规范

基准样张中有两个主要按钮：黄色按钮和红色按钮。后续按钮数量、文案长度和排列可以变化，但按钮必须保持实体按键感。

- 按钮宽度：主卡片内容宽度的 `68%-86%`。
- 按钮高度：画布高度的 `8%-13%`；基准样张填充区约 `203 px`。
- 按钮间距：按钮高度的 `35%-55%`。
- 按钮圆角半径：按钮高度的 `22%-28%`，基准范围 `44-58 px`。
- 按钮描边色：`#010101` 到 `#0C0C0C`。
- 按钮描边厚度：`8-12 px`。
- 按钮厚度阴影：
  - 水平偏移：`26-34 px`。
  - 垂直偏移：`26-36 px`。
  - 阴影颜色：`#0B0B0B` 到 `#111111`。
  - 模糊半径：`0-3 px`，推荐 `0 px`。
  - 阴影边缘必须硬，像实体按钮错位的黑色底层。
- 按钮内文字：
  - 字重：`800-900`。
  - 中文标题建议使用黑体类粗字，如 `Source Han Sans SC Heavy`、`Noto Sans CJK SC Black`、`PingFang SC Heavy`、`Microsoft YaHei UI Bold`。
  - 黄色按钮文字为黑色 `#010101`。
  - 红色按钮文字为白色 `#FFFFFF` 或 `#FFF8F2`。

## 窗口三色点

- 三色点直径：`46-48 px`。
- 第一个点位置：主卡片内边距左侧 `6%-8%`、顶部 `5%-7%` 附近。
- 三个点中心横向间距：圆点直径的 `1.65-1.8` 倍，基准约 `80-81 px`。
- 三色顺序：红、黄、绿。
- 圆点不可加文字，不可变成扁平小图标；它们负责提供“软件窗口”的语义。

## 字体与排版

- 主编号 `01`：
  - 字色：`#010101`。
  - 字重：`900`。
  - 字号应占画布高度约 `12%-20%`，根据内容复杂度动态调整。
  - 视觉位置：左中偏上，靠近画面主轴，不居中到页面正中。
- 主标题：
  - 字色：`#010101`。
  - 字重：`850-900`。
  - 中文字号应明显小于编号，但仍为画面第二层级。
  - 标题宽度不应超过主卡片内部宽度的 `82%`。
- 信息层级顺序必须清楚：编号最大，中文主题第二，按钮第三。
- 当没有编号时，主标题可以成为最大层级；但仍需保留粗黑体、大字号和高对比。
- 字间距：`0` 或接近 `0`，不要使用负字距。
- 避免细体、衬线体、手写体、书法体。

## 构图层级

推荐层级从底到顶：

1. 近白背景。
2. 右侧深色代码面板或软件窗口语境元素。
3. 主卡片硬阴影/厚度层。
4. 主卡片白色点阵面板与黑色粗描边。
5. 窗口三色点、编号、标题。
6. 黄色/红色实体按钮。
7. 鼠标指针、状态点等小型交互暗示。

右侧代码面板只能作为语境，不能超过画面宽度的 `12%-18%`，并且应被裁切在画布边缘；它不应与主卡片争夺注意力。

## 交互暗示元素

- 鼠标指针可出现，建议黑色填充、白色描边、轻投影。
- 指针尺寸建议为画布宽度的 `4%-6%`。
- 状态绿点直径建议 `38-44 px`。
- 小元素必须服务于“可操作界面”的感觉，不能变成装饰图案。

## 质感规则

- 立体感来自：粗黑描边、偏移阴影、按钮厚度、轻微模糊。
- 本风格的阴影必须偏硬：大面积极少模糊，边缘清楚，像实体纸板或厚 UI 层错位。
- 卡片边缘需要锐化：黑色描边外侧和阴影外侧都要有明确边界。
- 主卡片底色不是纯白空白面，需要低可见度点阵或类似系统网格纹理。
- 不使用玻璃拟态：避免透明磨砂、大面积背景模糊和半透明面板。
- 不使用低对比软拟态：边界必须清楚，黑色描边必须存在。
- 不使用过度真实材质：不要金属、皮革、木纹、纸张纹理。
- 不使用复杂渐变：按钮可有极轻微明暗变化，但主体应接近纯色。

## 生成检查清单

生成新图后，应检查：

- 画布是否为竖版，宽高比是否接近 `0.889`。
- 如果不是基准比例，是否仍保留同一套 token、粗描边、硬阴影和实体按钮感。
- 主卡片是否占画布宽度 `72%-88%`。
- 主卡片描边是否达到 `13-18 px` 的粗度。
- 主卡片是否有低可见度点阵底纹，且没有干扰文字。
- 黑色是否接近 `#010101`，不是深灰细线。
- 黄色按钮是否接近 `#FDC703`。
- 红色按钮是否接近 `#E03A10`。
- 卡片和按钮是否是硬阴影，模糊半径是否接近 `0 px`。
- 按钮是否有 `26-34 px` 的右下厚度偏移。
- 中文是否为超粗黑体，是否没有溢出按钮或卡片。
- 右侧代码面板是否只是背景语境，没有抢占主标题。
- 整体是否像“可点击的软件界面卡片”，而不是普通平面海报。

## 负向约束

不要生成以下风格：

- 扁平插画风。
- 玻璃拟态风。
- 低对比灰白软拟态。
- 赛博霓虹风。
- 复古纸张拼贴风。
- 极简细线 UI。
- 大面积紫蓝渐变或橙棕复古调色。
- 只有文字，没有实体按钮/窗口隐喻的普通知识海报。
